Getting It

<- PageTemplate

The Designer’s Perspective- >

The Easy Way

I finally got around to making a RubyGems gem package of PageTemplate available. This means that you can install the latest release of PageTemplate with one simple command:

gem install -r PageTemplate

You can always download your own copy of the gem file and install locally, if that’s your preference:

gem install -l PageTemplate

The Slightly Less Easy Way

Maybe you don’t use RubyGems, or you just don’t feel like mucking about with the gem system at all. That’s okay, I’ve got you covered. You only need to download and install the source package from Rubyforge.

Download

In order to save on the bandwidth for my home machine, I’ve opened a RubyForge account for PageTemplate. That means that you can always find the latest version of the PageTemplate package from my project download page:

http://rubyforge.org/projects/pagetemplate/

Once you’ve downloaded the latest version, unpack it into a temporary or source directory.

$ tar xfvzC PageTemplate-1_2_0.tar.gz ~/src/

Test

If you have Ruby 1.8, or Test::Unit installed on your machine, you can run the test cases that are used during development of PageTemplate.

$ ruby -w TC_PageTemplate.rb

If you get messages about any sort of failures, please let me know! I know that the tests run smoothly on my machine, but I have no idea how it’ll work on yours.

Install

Using install.rb

PageTemplate uses the standard ruby install.rb script for installation, which makes the process very easy:

ruby install.rb config
ruby install.rb setup
(su or sudo)
ruby install.rb install

Using Rake

If you have the Rake tool, you can use that to take care of testing and installing PageTemplate.

rake
sudo rake install
